# Code of Conduct

## Our Pledge

We want this project to be useful and welcoming for operators, developers, auditors, and contributors. Participants are expected to communicate respectfully, assume good intent, and keep discussions focused on improving the software.

## Expected Behavior

- Use clear, professional language.
- Respect different experience levels and operating contexts.
- Provide constructive technical feedback with enough detail to act on it.
- Avoid sharing secrets, private participant data, production databases, or regulated campaign details in public issues or pull requests.

## Unacceptable Behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing private information without permission.
- Encouraging unlawful lottery, gambling, payment, identity, or compliance workflows.
- Disruptive behavior that prevents maintainers or contributors from collaborating.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues or pull requests; or block users who violate this code of conduct. Security concerns should be reported through the process in [SECURITY.md](SECURITY.md), not through public comments.
